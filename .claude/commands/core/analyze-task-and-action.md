# Task 분석 후 플래닝 + 업무 수행

주로 Opus 가 Task 수행을 위해 사용하는 명령입니다. 가능한 한 사용자와 상호작용 하지 않고 한 번에 Task 를 완수하는 것을 목표로 합니다. 명령을 수행하다 도저히 해결이 안되는 지점이 발생할 경우에만 사용자에게 문의합니다.

1. 사용자가 전달한 Task 문서를 먼저 읽습니다.
2. 이 Task 를 완벽하고 완전하게 수행할 수 있도록 Task 와 관련된 문서나 코드가 있다면 이어서 읽고 분석합니다.
3. 이 업무를 수행하기 위해 ultrathink 하고 심도있게 사고해서 상세히 업무 플래닝을 합니다.
4. 차례대로 업무를 수행하되, 각 Phase 가 완료될 때마다 적절히 커밋합니다.
5. 구현이 완료되면 관련 테스트가 정상적으로 작성되었는지 확인하고 수행해 봅니다.
6. 모든 단계가 끝나면 Task 문서를 포함 관련 문서를 업데이트합니다.
