# Tasks 폴더 사용 / Task 티켓 생성 가이드

Caffeine 프로젝트는 99.9% AI Agent 가 작업을 하는 Workspace 로, 사용자가 Agent 에게 원활한 작업 지시를 하기 위해 Task 파일을 적극적으로 활용하고 있습니다. 따라서 Task 파일을 잘 활용할 수 있어야지 Agent 가 더 업무를 잘 수행해 낼 수 있어, Task 생성과 활용에 대해 아래와 같이 가이드합니다.

1. Task는 크게 Core / macOS 일감 타입으로 나뉜다.

2. 각 일감 타입에 따라 core / mac 폴더를 사용하고 그 안에 일감 티켓 파일을 만든다.

3. 티켓 파일은 [CORE-1] / [MAC-2] 와 같은 형식으로 타입 + 넘버링을 제일 먼저 하고, 그 뒤에 태스크 정보를 담도록 한다.
    - 예시 : [CORE-1] migrate-to-supabase.md
    - 예시 : [MAC-2] terraform-stage.md

4. 각 태스크 문서에는 새로운 피쳐나 개선사항, 버그 등을 AI Agent 가 개발하고 처리하기 위한 Task 내용이 들어가야 한다. 이 Task 문서는 Agent 가 온전하게, 독립적으로 업무를 잘 수행하기 위해 필요한 모든 것을 담고 있어야 한다.

5. 티켓의 진행 상태에 따라 파일 앞에 prefix 를 붙인다. 바로 당장 작업할 문서라면 [Next] prefix 를 붙이고, 이미 완료한 작업에 대한 문서를 작성하는 거라면 [Done] prefix 를 붙인다. 현재 작업 중인 사항에 대해 중간에 문서를 작성한는 거라면 [WIP] prefix 를 붙인다. 이외에도 적절한 prefix 가 있다면 사용해도 무방하다. [Done][BE-1] 형석으로 변경한다. [Done] [FE-30] 형태가 아니다.

6. Task 문서 안에는 AI Agent 가 독립적으로 업무를 잘 수행하기 위해 다음과 같은 요소를 포함해야 합니다.
    - 충분하고 완전한 작업 맥락
    - 작업 방향성
    - 주의사항
    - 명료하고 자세한 Task 완료 기준

7. 비즈니스와 밀접하게 관련된 Task 를 수행하는 경우 완료 기준을 제대로 지켰는지를 더 확실하게 보장하는 차원에서 TDD 기반으로 작업하는 것을 권장한다.
    - 모든 로직을 커버하기 보다는 중요한 비즈니스 로직 중심으로 테스트 스펙을 작성해놓으면 됩니다.
    - 코드단까지 디테일하게 작성할 필요는 없고 대략적인 스펙 정도면 됩니다. (given/when/then 형식을 사용해도 좋습니다)

8. 혹시 추가하는 Task 가 비즈니스상 규모가 크고 중요한 사항인데다 개발 로드맵 관점에서도 영향을 주는 건이라면, docs/PRD.md 문서에도 같이 업데이트한다.
